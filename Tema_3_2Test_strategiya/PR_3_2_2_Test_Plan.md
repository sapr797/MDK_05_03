## Теоретическая справка

### Разница между тест-стратегией и тест-планом

| Характеристика | Тест-стратегия | Тест-план |
|:---|:---|:---|
| **Уровень** | Высокий (стратегический) | Детальный (тактический) |
| **Длительность** | На весь проект | На конкретный релиз/спринт |
| **Аудитория** | Руководство, заказчик, PM | Команда тестирования, разработчики |
| **Изменяемость** | Редко | Перед каждым релизом |
| **Что содержит** | Подходы, принципы, инструменты | Даты, задачи, конкретные тесты |
| **Ответственный** | Тест-менеджер / Lead QA | QA-инженер / Тестировщик |

## Объект тестирования

В качестве учебного проекта возьмём **«Систему управления задачами (Task Manager) на Python + FastAPI»** (аналог Laravel, но на Python).

### Функциональные требования

| ID | Функция | Описание |
|:---|:---|:---|
| F-01 | Регистрация | Пользователь может создать аккаунт (email, пароль, имя) |
| F-02 | Аутентификация | Вход по email и паролю, JWT-токены, восстановление пароля |
| F-03 | Создание задачи | Заголовок (обяз.), описание (опц.), дедлайн, приоритет (низкий/средний/высокий) |
| F-04 | Редактирование задачи | Изменение любых полей, логирование изменений |
| F-05 | Удаление задачи | Мягкое удаление (в корзину) + восстановление |
| F-06 | Список задач | Фильтрация по статусу, приоритету, дедлайну; поиск по заголовку |
| F-07 | Назначение исполнителя | Только для автора задачи или админа |
| F-08 | Комментарии | Добавление комментариев к задаче |
| F-09 | Роли | Пользователь, Администратор (может управлять всеми задачами) |
| F-10 | Отчёты | Экспорт списка задач в CSV/PDF |

### Нефункциональные требования

| ID | Требование | Значение |
|:---|:---|:---|
| N-01 | Производительность | Время ответа API < 200 мс (95% запросов) |
| N-02 | Нагрузка | Выдержка 1000 одновременных пользователей |
| N-03 | Безопасность | Защита от XSS, CSRF, SQL-инъекций |
| N-04 | Доступность | 99.9% времени |
| N-05 | Совместимость | Chrome, Firefox, Safari (последние 2 версии) |

# .github/workflows/test.yml

name: Run Tests

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    
    services:
      postgres:
        image: postgres:15
        env:
          POSTGRES_PASSWORD: testpass
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
        ports:
          - 5432:5432
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'
      
      - name: Install dependencies
        run: |
          pip install -r requirements.txt
          pip install pytest pytest-cov
      
      - name: Run unit tests
        run: pytest tests/unit/ --cov=app --cov-report=xml
      
      - name: Run integration tests
        run: pytest tests/integration/ --cov=app --cov-append
      
      - name: Run API tests
        run: pytest tests/api/ --cov=app --cov-append
      
      - name: Upload coverage
        uses: codecov/codecov-action@v3
