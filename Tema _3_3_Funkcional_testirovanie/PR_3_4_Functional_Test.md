## Теоретическая справка (5 минут)

### Чек-лист vs Сценарий vs Тест-кейс

| Документ | Назначение | Уровень детализации | Кто использует |
|:---|:---|:---|:---|
| **Чек-лист** | Быстрая проверка (галочки) | Низкий (одна строка = одна проверка) | Любой тестировщик |
| **Сценарий (use case)** | Описание взаимодействия пользователя с системой | Средний (описание потока) | Аналитик, QA |
| **Тест-кейс** | Детальная инструкция для выполнения теста | Высокий (шаг за шагом) | QC-инженер, автоматизатор |

markdown
# Use Case: Создание задачи

**ID:** UC-01
**Название:** Создание новой задачи
**Актор:** Авторизованный пользователь
**Приоритет:** High

**Предусловия:**
- Пользователь авторизован в системе
- Пользователь находится на странице списка задач (/tasks)

**Основной поток (Happy Path):**
1. Пользователь нажимает кнопку «Создать задачу»
2. Система открывает форму создания задачи (/tasks/create)
3. Пользователь заполняет поля:
   - Заголовок (например, «Купить продукты»)
   - Описание (например, «Молоко, хлеб, яйца»)
   - Дедлайн (выбирает дату из календаря)
   - Приоритет (выбирает «Средний»)
4. Пользователь нажимает кнопку «Сохранить»
5. Система валидирует данные:
   - Заголовок не пустой
   - Заголовок не длиннее 255 символов
   - Дедлайн не в прошлом (если указан)
6. Система сохраняет задачу в БД
7. Система редиректит на страницу списка задач (/tasks)
8. Система отображает сообщение «Задача успешно создана»
9. Новая задача появляется в списке (первая строка или с учётом сортировки)

**Альтернативные потоки:**

**A1. Обязательное поле «Заголовок» пустое:**
- 4a. Пользователь не заполнил заголовок, нажимает «Сохранить»
- 5a. Система показывает ошибку «Поле заголовок обязательно»
- 6a. Задача не сохраняется, пользователь остаётся на форме

**A2. Дедлайн в прошлом:**
- 3a. Пользователь выбирает дату вчерашнего дня
- 5a. Система показывает ошибку «Дедлайн не может быть в прошлом»
- 6a. Задача не сохраняется

**A3. Отмена создания:**
- 3a. Пользователь нажимает кнопку «Отмена» (не «Сохранить»)
- 4a. Система редиректит на /tasks без сохранения задачи
- 5a. Сообщение не отображается

**A4. Переполнение заголовка (>255 символов):**
- 3a. Пользователь вводит заголовок длиной 300 символов
- 5a. Система показывает ошибку «Заголовок не может превышать 255 символов»
- 6a. Задача не сохраняется

**Постусловия:**
- При успешном создании: в БД появляется запись задачи
- При отмене или ошибке: БД не изменяется
Задание: Добавьте ещё 2 альтернативных потока (подумайте, что ещё может пойти не так).

Напишите автоматический UI-тест для сценария «Создание задачи» с использованием Laravel Dusk или Selenium.

php
// Пример для Laravel Dusk
public function testUserCanCreateTask()
{
    $this->browse(function (Browser $browser) {
        $browser->loginAs(1) // авторизоваться
                ->visit('/tasks/create')
                ->type('title', 'Автотест задача')
                ->type('description', 'Создано автоматически')
                ->press('Сохранить')
                ->assertPathIs('/tasks')
                ->assertSee('Автотест задача');
    });
}
