## 1. Что такое регрессионное тестирование?

### 1.1 Определение

**Регрессионное тестирование (Regression Testing)** — это повторный запуск ранее выполненных тестов для проверки, что новые изменения в коде не нарушили существующую функциональность.
┌─────────────────────────────────────────────────────────────────────────────┐
│ ЧТО ТАКОЕ РЕГРЕССИЯ? │
├─────────────────────────────────────────────────────────────────────────────┤
│ │
│ Исходный код После изменений │
│ ┌─────────────────┐ ┌─────────────────┐ │
│ │ Функция А ✅ │ │ Функция А ❌ │ ← РЕГРЕССИЯ! │
│ │ Функция Б ✅ │ ──────► │ Функция Б ✅ │ │
│ │ Функция В ✅ │ изменения │ Функция В ✅ │ │
│ └─────────────────┘ └─────────────────┘ │
│ │
│ Регрессия — это когда новый код ломает СТАРУЮ функциональность │
│ │
└─────────────────────────────────────────────────────────────────────────────┘

text

### 1.2 Основные понятия

| Термин | Определение |
|:---|:---|
| **Регрессия** | Ошибка, возникшая в ранее работавшей функциональности после изменений |
| **Регрессионный тест** | Тест, проверяющий, что изменения не сломали существующее поведение |
| **Регрессионный набор** | Набор тестов, регулярно запускаемых для обнаружения регрессий |
| **Выборочная регрессия** | Запуск только тестов, связанных с изменённым кодом |

### 3.2 Частота запуска в зависимости от этапа

| Этап разработки | Частота регрессии | Объём |
|:---|:---|:---|
| **Разработка (локально)** | После каждого коммита | Выборочная (связанные тесты) |
| **CI (Pull Request)** | При каждом PR | Быстрая (smoke + связанные) |
| **Стейджинг** | Несколько раз в день | Полная автоматизированная |
| **Перед релизом** | Перед каждым релизом | Полная (авто + ручная) |

### 4.2 Стратегии выбора тестов

| Стратегия | Описание | Когда использовать |
|:---|:---|:---|
| **Полная регрессия** | Все тесты | Перед релизом |
| **Выборочная регрессия** | Тесты на изменённые модули | При каждом PR |
| **Приоритетная регрессия** | Только критические тесты | Smoke-тестирование |
| **Модификационная регрессия** | Тесты, связанные с изменениями | При точечных правках |

### 4.3 Анализ влияния изменений

```python
# Пример: определение связанных тестов
def get_affected_tests(changed_files: List[str]) -> List[str]:
    """
    Определяет, какие тесты нужно запустить на основе изменённых файлов.
    """
    affected_tests = []
    
    # Карта соответствия "файл → тесты"
    mapping = {
        "user_manager.py": ["test_user_creation", "test_user_deletion", "test_user_update"],
        "auth.py": ["test_login", "test_logout", "test_password_reset"],
        "payment.py": ["test_checkout", "test_refund"],
    }
    
    for file in changed_files:
        if file in mapping:
            affected_tests.extend(mapping[file])
    
    return list(set(affected_tests))  # уникальные

# regression_suite.py

import pytest
from typing import List, Dict


class RegressionTestSuite:
    """Класс для управления регрессионными тестами."""
    
    def __init__(self):
        self.critical_tests = []
        self.all_tests = []
    
    def add_critical_test(self, test_name: str, test_func):
        """Добавляет критический тест (запускается всегда)."""
        self.critical_tests.append((test_name, test_func))
    
    def add_test(self, test_name: str, test_func, priority: str = "medium"):
        """Добавляет тест с приоритетом."""
        self.all_tests.append((test_name, test_func, priority))
    
    def run_smoke_regression(self) -> Dict[str, bool]:
        """Быстрая регрессия (только критическое)."""
        results = {}
        for name, func in self.critical_tests:
            try:
                func()
                results[name] = True
            except AssertionError:
                results[name] = False
        return results
    
    def run_full_regression(self) -> Dict[str, bool]:
        """Полная регрессия (все тесты)."""
        results = {}
        for name, func, priority in self.all_tests:
            try:
                func()
                results[name] = True
            except AssertionError:
                results[name] False
        return results


# Пример использования
regression = RegressionTestSuite()

@regression.add_critical_test("test_login")
def test_login():
    assert login("user", "pass") is True

@regression.add_critical_test("test_checkout")
def test_checkout():
    assert checkout() is True

# Запуск быстрой регрессии
results = regression.run_smoke_regression()

6. Стратегии регрессионного тестирования
6.1 Сравнение стратегий
Стратегия	Время выполнения	Покрытие	Риски пропуска
Полная регрессия	Высокое	100%	Низкий
Выборочная регрессия	Среднее	30-70%	Средний
Приоритетная регрессия	Низкое	10-20%	Высокий
Модификационная регрессия	Низкое	Переменное	Средний
6.2 Матрица выбора стратегии
Сценарий	Рекомендуемая стратегия
Исправление бага в критическом модуле	Полная регрессия
Добавление новой функции	Выборочная + новая
Изменение документации	Только smoke
Рефакторинг без изменения API	Полная регрессия
Срочный hotfix в продакшн	Приоритетная (критическое)
Ночная сборка	Полная регрессия (все тесты)
7. Метрики регрессионного тестирования
7.1 Ключевые метрики
Метрика	Формула	Цель
Процент регрессий	(число регрессий / всего багов) × 100%	< 10%
Время выполнения регрессии	Время в минутах	< 30 мин
Стабильность тестов	(число стабильных / всего) × 100%	> 95%
Покрытие критических сценариев	(покрыто / всего критических) × 100%	100%

# regressions_tracker.py

class RegressionTracker:
    """Трекер регрессий."""
    
    def __init__(self):
        self.regressions: List[Dict] = []
    
    def log_regression(self, test_name: str, change_type: str, author: str):
        """Логирует найденную регрессию."""
        self.regressions.append({
            "test": test_name,
            "change_type": change_type,  # bugfix, feature, refactor
            "author": author,
            "detected_at": datetime.now()
        })
    
    def get_regression_rate(self, total_bugs: int) -> float:
        """Процент регрессий среди всех багов."""
        return len(self.regressions) / total_bugs * 100
    
    def get_regressions_by_author(self) -> Dict[str, int]:
        """Статистика регрессий по авторам."""
        stats = {}
        for reg in self.regressions:
            author = reg["author"]
            stats[author] = stats.get(author, 0) + 1
        return stats
